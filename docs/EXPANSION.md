# 추가 확장 계획 — EduWatch 실서비스 로드맵

> 본 문서는 공모전 프로토타입 이후, 실서비스로 확장할 때의 계획을 정리한 문서입니다.
> 심사위원에게 "향후 비전"을 설명할 때 참고합니다.

## 1. 에듀 포인트 보상 시스템

### 개요
학생의 학업 의지를 높이기 위한 포인트 보상 시스템.
학부모가 목표 집중률과 보상 포인트를 설정하고, 학생이 기준을 달성하면 에듀 포인트를 획득한다.
포인트의 실물 보상 교환(용돈, 선물 등)은 가정 내에서 자율적으로 진행.

### 프로토타입 구현 (공모전 범위)
- 학부모 설정 화면: 목표 집중률(%), 세션당 보상 포인트, 주간 보너스 조건
- 학생 화면: 강의 중 목표 달성 진행률 표시, 누적 포인트 및 획득 내역
- 포인트 충전: 시뮬레이션 (버튼 클릭으로 즉시 충전, 실결제 없음)

### 학부모 설정 항목
- 목표 집중률: 슬라이더로 50%~95% 범위 설정 (기본값 70%)
- 세션당 보상: 10P~500P 범위 설정 (기본값 100P)
- 주간 보너스: 주 N회 달성 시 추가 포인트 (기본값: 주 5회 달성 시 500P)
- 포인트 예산: 학부모가 충전하는 총 포인트 (기본값 10,000P)

### 학생 화면 반영
- 강의 중 집중도 위젯에 "목표: 80% / 현재: 73%" 실시간 표시
- 대시보드에 누적 포인트, 최근 획득 내역, 주간 목표 달성 현황
- 목표 달성 시 축하 애니메이션 (간단한 모달)

### 데이터 모델
```
EduPoint {
  parentId,
  balance,
  settings: {
    targetRate,
    rewardPerSession,
    weeklyBonusCount,
    weeklyBonusReward
  }
}

PointHistory {
  studentId, date, sessionId,
  earned, reason("세션달성" | "주간보너스"),
  balanceAfter
}
```

### 부정행위 방어
- AI 모델이 표정/자세를 종합 분석하므로, 단순히 화면을 바라보는 것만으로 "집중+흥미" 판정을 받기 어려움
- 동일 자세 장시간 유지 시 "비집중+차분" 또는 "졸음"으로 분류될 가능성 높음
- 향후 확장: 랜드마크 움직임 패턴 분석으로 부자연스러운 행동 감지

## 2. 실결제 포인트 충전 (PG사 연동)

### 현재 (프로토타입)
- 학부모가 "충전" 버튼 클릭 시 실결제 없이 포인트 즉시 추가

### 실서비스 확장 시
- PG사(토스페이먼츠, NHN KCP 등) API 연동
- 결제 흐름: 학부모 충전 요청 → PG사 결제 화면 → 카드/계좌 결제 → 성공 콜백 → 포인트 반영
- 우리 서버는 카드 정보를 직접 다루지 않음 (PG사가 중간 처리)
- 최소 충전 단위, 환불 정책 등 부가 정책 필요

## 3. 멀티모달 AI 모델 확장

### 현재 (프로토타입)
- 영상 전용 모델 (MobileNet V3 Large, F1: 0.97)
- 웹캠 영상만으로 학습태도 5클래스 분류

### 실서비스 확장 시
- 멀티모달 모델 (MobileNet V3 Large + MLP + Tree-LSTM, F1: 0.99) 적용
- 영상 + 환경소음 + 뇌파 데이터를 결합한 고정밀 분류
- 뇌파 분석: EEG 헤드밴드(Muse 등) 연동 필요 → 전문 교육기관/연구소 대상
- 환경소음 분석: 마이크 입력으로 주변 소음 수준 반영 (비교적 구현 용이)

## 4. 인강 플랫폼 연동

### 현재 (프로토타입)
- 자체 웹앱에 샘플 강의 영상 + 자막 내장
- Claude API로 자막 기반 강좌 콘텐츠 분석 + RAG 맞춤형 리포트 생성 구현 완료

### 실서비스 확장 시
- 인강 플랫폼 API 연동으로 수강 중인 강좌 + 자막 자동 수집
- 대량 강좌 자막 자동 분석 파이프라인 (배치 처리)
- 크롬 확장 프로그램으로 기존 인강 사이트(메가스터디, 대성마이맥 등)에서 자동 활성화
- LMS(학습관리시스템) 연동으로 학교/학원 단위 도입 가능
- 자막 미제공 강좌: Whisper API(STT)로 음성→텍스트 변환 후 분석

## 5. AI 퀴즈 자동 생성 (DeepTutor 참고)

### 참고
- [DeepTutor](https://github.com/HKUDS/DeepTutor) (홍콩대 HKUDS Lab, Apache 2.0)
- Deep Question 모듈에서 아이디어 차용: 지식 기반에서 퀴즈를 자동 생성하는 에이전트 파이프라인

### 개요
세션 리포트에서 **집중도가 낮았던 강의 구간**의 핵심 내용을 퀴즈로 자동 생성하여, 학생이 해당 내용을 실제로 이해했는지 확인하는 기능.
학부모에게는 "퀴즈 정답률"이라는 객관적 학습 성과 지표를 추가로 제공.

### 프로토타입 구현 (공모전 범위 — Claude API 활용)
- 세션 리포트 하단에 "복습 퀴즈" 섹션 추가
- Claude API에 집중도 낮은 구간의 자막 + segments 전달
- 객관식 3~5문제 자동 생성 (문제 + 선택지 4개 + 정답 + 해설)
- 학생이 퀴즈 풀면 정답률 표시 + 오답 해설 제공

### Claude API 프롬프트 설계
```
입력:
- 강의 제목, 과목
- 집중도 낮은 구간의 자막 텍스트
- 해당 구간의 주제/키워드 (segments에서 추출)

요청:
"다음 강의 내용에서 핵심 개념을 확인하는 객관식 퀴즈 3문제를 생성해주세요.
각 문제는 { question, options[4], answer, explanation } 형식의 JSON으로 작성해주세요."
```

### 데이터 모델
```
Quiz {
  sessionId,
  lectureId,
  questions: [
    {
      question: "삼차방정식의 인수분해에서 조립제법을 사용하는 이유는?",
      options: ["a", "b", "c", "d"],
      answer: 0,
      explanation: "조립제법은 삼차 이상의 다항식에서..."
    }
  ],
  results: {
    score: 2,
    total: 3,
    completedAt: "2026-04-07T15:30:00.000Z"
  }
}
```

### 학부모 리포트 반영
- 세션 리포트에 퀴즈 정답률 표시 ("복습 퀴즈: 2/3 정답")
- 주간 리포트에 퀴즈 평균 정답률 추이
- 오답이 많은 주제 자동 정리 → 취약 단원 파악

### 실서비스 확장 시 (DeepTutor 에이전트 파이프라인 참고)
- 단순 객관식 → 주관식, 풀이 과정 평가 (Deep Solve 개념 차용)
- 퀴즈 난이도 자동 조절 (학생 수준에 맞춤)
- 오답 기반 추가 학습 자료 추천 (Guided Learning 개념 차용)
- 장기 학습 메모리: 학생별 취약 단원 프로필 축적 → 맞춤 복습 스케줄

## 6. AI 기반 학습 가이드 (DeepTutor Guided Learning 참고)

### 개요
집중도가 낮았던 구간에 대해 "이 부분을 다시 학습하세요"가 아닌,
단계별 인터렉티브 학습 가이드를 자동 생성하여 제공하는 기능.

### 실서비스 확장 시
- 취약 구간 자동 식별 → 3~5단계 학습 계획 생성 (Claude API)
- 각 단계별 핵심 개념 설명 + 예제 + 확인 퀴즈
- 학습 진도 추적 (몇 단계까지 완료했는지)
- 학부모에게 "보충 학습 진행률" 리포트 제공

## 7. 학습자 프로필 및 장기 메모리 (DeepTutor Memory 참고)

### 개요
DeepTutor의 2계층 메모리 시스템(학습 요약 + 학습자 프로필)을 차용하여,
장기적으로 학생의 학습 특성을 축적하고 맞춤형 코칭에 활용.

### 실서비스 확장 시
- 학습 요약 (Summary): 주간/월간 학습 이력 자동 요약 (Claude API)
- 학습자 프로필 (Profile): 선호 학습 시간대, 과목별 강약점, 집중 패턴
- RAG 리포트 생성 시 프로필 데이터를 컨텍스트로 포함 → 더 개인화된 분석
- 학부모 대시보드에 "자녀 학습 특성 분석" 카드 추가

## 8. 추가 확장 아이디어

### 교강사 대시보드
- 수강생 전체의 집중도 통계를 교강사에게 제공
- 어떤 구간에서 집중도가 떨어지는지 파악 → 강의 개선에 활용

### 학습 패턴 AI 분석
- 장기간 데이터 축적 후 개인별 최적 학습 시간대/과목 순서 추천
- "수학은 오전에, 영어는 오후에 집중도가 높습니다" 같은 맞춤 코칭

### 또래 비교 (익명)
- 같은 학년/과목 수강생의 평균 집중률과 비교
- 개인정보 없이 통계치만 활용 (상위 30% 등)
