# 기획 문서 — EduWatch: AI 학습태도 모니터링 & 학부모 리포트 서비스

## 1. 프로젝트 개요
- **공모전**: 2026 KIT 바이브코딩 공모전
- **주제**: AI활용 차세대 교육 솔루션
- **마감**: 2026.04.13(월)
- **서비스명**: EduWatch

## 2. 문제 정의 (Pain Point)
인터넷 강의 시장이 급성장하면서, 학생이 혼자 인강을 듣는 시간이 늘어나고 있다.
하지만 학부모는 자녀가 실제로 집중해서 수업을 듣고 있는지 확인할 방법이 없다.
- 학생: 인강을 틀어놓고 딴짓, 졸음, 핸드폰 사용 등
- 학부모: 자녀의 학습 상태를 알 수 없는 불안감
- 교강사: 수강생의 실시간 반응/집중도 파악 불가

## 3. 솔루션
웹캠으로 학습자의 표정/자세를 AI가 실시간 분석하여 집중도를 측정하고,
탭 이탈 감지(Page Visibility API)로 학습 이탈을 추적하며,
강의 자막 기반 콘텐츠 분석 + Claude API RAG를 활용해
"어떤 내용에서 집중도가 떨어졌는지"까지 분석하는 맞춤형 학습 리포트를
학부모에게 자동으로 제공하고,
집중도가 낮았던 구간의 핵심 내용을 AI 퀴즈로 자동 생성하여
학생의 실제 이해도를 확인하는 웹 서비스.

## 4. AI 모델 상세 (AI Hub 데이터셋 기반)

### 사용 데이터셋
- AI Hub "학습태도 및 성향관찰 데이터" (데이터셋 번호 71715)
- 구축: 벨텍소프트 컨소시엄 / 2023년
- 라이선스: Apache License 2.0 (상업적 이용 가능)

### 모델 파이프라인 (4단계)
```
[웹캠 영상] → [YOLO v8] → [MobileNet V2] → [MobileNet V3 Large] → 분류 결과
              얼굴 검출    랜드마크 68점     학습태도 5클래스 분류
```

| 단계 | 모델 | 역할 | 비고 |
|------|------|------|------|
| 1 | YOLO v8 (CSPDarknet53) | 얼굴 바운딩 박스 검출 | |
| 2 | MobileNet V2 | 얼굴 랜드마크 68점 추출 | TensorFlow.js face-landmarks-detection |
| 3 | MobileNet V3 Large | 학습태도 5클래스 분류 (영상 전용) | F1: 0.97 |
| 4 | MobileNet V3 Large + MLP + Tree-LSTM | 멀티모달 분류 (영상+소음+뇌파) | F1: 0.99 (→ EXPANSION.md) |

### 5가지 분류 클래스
1. 집중 + 흥미로움
2. 집중 + 차분함
3. 집중하지 않음 + 차분함
4. 집중하지 않음 + 지루함
5. 졸음

### 우리 프로젝트에서 사용할 모델
- **영상 전용 모델 (F1: 0.97)** 만 사용
- 뇌파 모델은 전문 장비(EEG 헤드밴드)가 필요하므로 제외
- 웹캠만으로 97% 정확도 → 일반 사용자 접근성 확보

### 모델 학습 환경 (참고)
- Python 3.10.13, TensorFlow-GPU 2.9.3
- MobileNet V3 Large를 ImageNet pretrained 모델로 전이학습
- 학습 조건: epoch 300, batch 256, lr 0.0001, optimizer Adam
- 학습 데이터: 1,448,070개 얼굴 이미지 (USER_ID별 클래스별)

## 5. 강좌 콘텐츠 분석 & RAG 리포트 시스템

### 개요
강의 자막을 Claude API로 사전 분석하여 구간별 주제를 추출하고,
세션 리포트 생성 시 집중도 데이터와 강좌 콘텐츠를 결합(RAG)하여
"어떤 내용에서 집중도가 떨어졌는지" 구체적으로 분석하는 맞춤형 리포트를 생성한다.

### 강좌 콘텐츠 사전 분석 파이프라인
```
[강의 자막 파일 (SRT/VTT)] → 자막 파싱 (타임스탬프 + 텍스트)
  → Claude API로 구간별 주제 분석
  → 구조화된 JSON으로 DB 저장
```

분석 결과 JSON 예시:
```json
{
  "lectureId": "lec-1",
  "title": "미적분 기초",
  "segments": [
    { "start": "00:00", "end": "05:30", "topic": "이차방정식 복습", "keywords": ["이차방정식", "근의 공식"] },
    { "start": "05:30", "end": "12:00", "topic": "삼차방정식 개념 도입", "keywords": ["삼차방정식", "인수분해"] },
    { "start": "12:00", "end": "18:00", "topic": "삼차방정식 풀이법", "keywords": ["조립제법", "근과 계수"] },
    { "start": "18:00", "end": "25:00", "topic": "연습 문제 풀이", "keywords": ["예제", "풀이 과정"] }
  ]
}
```

### RAG 기반 맞춤형 리포트 생성
```
[세션 종료 시]
  학생 집중도 타임라인 + 탭 이탈 기록
  + 해당 강좌의 콘텐츠 분석 JSON
  → Claude API에 함께 전달 (RAG)
  → 맞춤형 분석 리포트 생성
```

기존 규칙 기반 팁:
- "15분~20분 구간에서 집중도가 하락했습니다"

RAG 기반 맞춤형 분석:
- "삼차방정식 풀이법을 설명하는 12:00~18:00 구간에서 집중도가 62%로 하락했습니다. 이전 구간(삼차방정식 개념)에서는 85%를 유지한 것으로 보아, 개념 이해는 되었으나 풀이 과정에서 어려움을 느낀 것으로 보입니다. 조립제법 관련 보충 학습을 권장합니다."

### 기술 상세
- **자막 파싱**: SRT/VTT 파일에서 타임스탬프 + 텍스트 추출 (Node.js에서 직접 파싱)
- **Claude API**: Anthropic Claude API (claude-sonnet 모델 사용, 비용 효율적)
- **비용**: 자막 텍스트(30분 강의 = 수천 토큰) + 리포트 생성 요청 → 건당 수 센트
- **API 예산**: $300 (프로토타입 + 데모 운영에 충분)
- **사전 분석 시점**: 강좌 등록 시 1회 실행, 결과를 lectures.json에 저장

## 6. 온디바이스 AI 아키텍처

### 핵심 설계 원칙
AI 추론은 학생 기기(브라우저)에서 실행, 서버로는 분류 결과만 전송.
→ 개인정보 보호 + 서버 부담 최소화

```
[학생 브라우저 - 온디바이스]                    [서버]
 웹캠 → 프레임 캡처 (1~3초 간격)
 → TensorFlow.js face-landmarks-detection (얼굴 검출 + 랜드마크 추출)
 → MobileNet V3 집중도 분류
 → 분류 결과(숫자)만 전송 ──────────→  결과 저장 & 집계
                                              리포트 생성
                                              학부모 대시보드 제공
```

### 데이터 흐름 (1.5시간 강의 기준)
- 3초 간격 분석 → 약 1,800개 분류 결과 생성
- 서버로 전송되는 데이터: 타임스탬프 + 분류값(1~5) → 세션당 수십 KB
- 영상/음성 데이터는 서버로 전송하지 않음

### 브라우저 AI 구현
- TensorFlow.js로 통합 구현 (별도 MediaPipe 패키지 불필요)
  - @tensorflow/tfjs: 핵심 런타임
  - @tensorflow-models/face-landmarks-detection: 얼굴 검출 + 랜드마크 추출 (내부적으로 MediaPipe와 동일 모델 사용, 성능 차이 없음)
- 실행 환경 자동 감지: WebGL(GPU) → WASM(CPU) 폴백
  - 전용 GPU 불필요, 내장 그래픽(Intel UHD 등)으로 충분
  - 태블릿/스마트폰 브라우저에서도 동작
- 모델 파일 크기: best.h5 약 20~30MB (MobileNet 경량 모델)
- 모델 변환: Python에서 h5 → TensorFlow.js 포맷 1회 변환 필요

## 7. 사용자 흐름 (UX)

### 학생 화면
```
1. EduWatch 웹앱 로그인
2. 수강 가능한 강좌 목록에서 강좌 선택
3. "카메라 권한 허용하시겠습니까?" 팝업 → 허용
4. 강의 영상 재생 + 우측 상단 집중도 실시간 표시
   (작은 위젯: 현재 상태 아이콘 + 집중률 %)
5. 탭 이탈 감지: 학생이 다른 탭/앱으로 전환하면 이탈 시간 기록
6. 강의 종료 → 세션 리포트 자동 생성 (집중도 분석 + 탭 이탈 통계)
7. 복습 퀴즈: 집중도 낮은 구간의 핵심 내용 객관식 퀴즈 (Claude API 자동 생성)
```

### 학부모 화면
```
1. EduWatch 웹앱 로그인 (학부모 계정)
2. 자녀의 학습 대시보드
   - 오늘의 학습 요약 (총 학습시간, 평균 집중률, 탭 이탈 횟수)
   - 일간/주간 집중도 추이 그래프
   - 세션별 타임라인 (집중/이탈 구간 시각화)
   - 졸음 빈도, 집중 저하 시간대 분석
3. AI 코칭 메시지 (규칙 기반)
   - "오후 2시 이후 집중력이 급격히 떨어집니다"
   - "수학 과목 집중도가 영어보다 20% 낮습니다"
4. RAG 맞춤형 분석 (Claude API)
   - "삼차방정식 풀이법 구간(12:00~18:00)에서 집중도가 62%로 하락했습니다"
   - "개념 이해는 되었으나 풀이 과정에서 어려움을 느낀 것으로 보입니다"
```
